# Implementation Plan: Ramp Resource LP — Ground Handling Worker Scheduling

**Branch**: `001-ramp-resource-lp` | **Date**: 2026-05-02 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-ramp-resource-lp/spec.md`

## Summary

Two-stage LP system that converts a flight schedule (plus optional actual movement data) into the minimum number of ground-handling worker shifts needed to cover every operating hour at a Finavia airport. Stage 1 (`compute_demand`) produces an hourly worker demand curve from per-slot arrival and departure counts, aircraft-type staffing standards, and delay adjustments (delay flags, slot-level actuals, or per-flight tolerance-window reclassification). Stage 2 (`schedule_shifts`) solves the textbook shift-start LP minimising total workers while satisfying all hourly coverage constraints. The integration layer (`business_problems/ramp_resource_lp.py`) loads the EFHK CSV, converts UTC timestamps to Helsinki local time via `ZoneInfo("Europe/Helsinki")`, maps ICAO codes to LP categories, extracts embedded actual times to build `FlightMovementInput` records by default, and calls into `src/lp/`.

## Technical Context

**Language/Version**: Python 3.10+ (required by constitution)
**Primary Dependencies**: `ortools` (GLOP LP solver) — declared as `lp` optional group in `pyproject.toml`; standard library only (`dataclasses`, `typing`, `math`, `zoneinfo`, `enum`)
**Storage**: N/A — in-memory computation, no persistent state
**Testing**: `pytest` (mandatory per constitution III); `tests/lp/` mirrors `src/lp/` hierarchy
**Target Platform**: Python library; callable from scripts, notebooks, or API layers
**Project Type**: Library (src module + business_problems integration layer)
**Performance Goals**: Full-day demand curve computation in < 30 seconds (SC-001); LP solve time negligible at 18-slot scale
**Constraints**: Zero extra serialisation dependencies; all public types use `dataclasses` / `TypedDict` with full type hints; ceiling rounding post-solve (not MIP)
**Scale/Scope**: 18 hourly slots per operating day; 3 aircraft categories; ~400 flight movements per day (EFHK reference)

**Structure Decision**: Single-project layout (Option 1). The `src/lp/` module is the LP solver core; `src/utils/efhk_loader.py` is the airport-specific data-loading layer (constitution rule 5: data-loading utilities belong in `src/utils/`). This keeps ICAO mapping, timezone conversion, and CSV parsing concerns out of the solver module.
